ioHub Documentation and API Specification¶

ioHub is a Python package that provides a device input event monitoring and saving framework for use in psychology or cognitive neuroscience experiment research. ioHub is not a complete experiment design and runtime API, but is instead intended to be used with existing Python experiment runtime packages, primarily the most excellent PsychoPy .

ioHub supports the following high level functionality:

1. Monitoring of input events from experiment input devices such as the keyboard, mouse, analog to digital converters, XInput compatible gamepads, and eye trackers via a common eye tracking interface that provides the same user level API regardless of the eye tracking hardware used. 1. Device event monitoring is done completely independently from the experiment runtime graphics environment and allows for device inputs to be captured system wide, not just for events targeted to the stimulus presentation window. In fact, no graphical window in needed at all to collect inputs from the majority of supported devices. 1. Device input monitoring is run as a seperate process from the experiment runtime itself, called the ioHub Server Process. This allows for input event monitoring and device event callback processing to occur very quickly and regardless of what state the experiment runtime process is in (i.e. even when the experiment runtime process is blocked and would not be able to monitor for new events itself). 1. Device inputs can be saved by the ioHub Server Process for post hoc analysis. Assuming a multicore CPU is being used, in general all device events can be saved during the experiment without effecting the performance of the experiment runtime logic itself. 1. Events can be accessed by the experiment script during runtime either at a global, device independent manner, or for a specific device type alone. 1. Text Messages can be registered with ioHub Server, allowing easy integration of experiment runtime events (such as important stimuli onsets, etc.) 1. A common time base is provided for all device input events, and between the experiment runtime and ioHub Server processes, making it easy to syncronize experiment and device input events. Events are time stamped when it is not done by the source device itself, or existing device event time stamps are converted to the common ioHub time base. 1. Device inputs are converted into common event types based on device type, regardless of the underlying supported hardware.
